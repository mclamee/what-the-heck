# What The Heck 🫷

[English](README.md) · **中文**

**一句话指令，让你的 AI 别再打太极，直接说出它真正的判断。**

> `/what-the-heck`

一个 [Claude Code](https://docs.claude.com/en/docs/claude-code) 技能，专治最强模型
——尤其是 **Claude Opus 4.8**——最让人上头的毛病：它*有*结论，却就是不肯拍板。你要
的是一个答案，拿到的却是一份排版工整的编号菜单，挑选的活儿被悄悄甩回给了你。

这个技能就是用户那句「**你这是在干嘛（what the heck are you doing）？**」——一旦说出
口，模型就被强制做一次诚实的自我审计：说出它唯一真实的推荐，并且（在可逆、低风险时）
**直接动手**。

---

## 它解决什么问题 —— "不说人话"

模型越聪明，回避得越礼貌。你要一个决定，换来的是：

```
你想怎么推进？
  1. 现在合并
  2. 等配套 PR
  3. 先 hold 一下回头再看
```

——而这个 PR **早已 approved**，指向的 commit **早已 merged**。答案只有一个。模型
心知肚明。这个三选项外壳不是谦让，而是 **procedural politeness as cost-shifting
（用程序性礼貌转嫁成本）**：看着像尊重你的自主权，实则是拒绝承担自己的判断，把决策
成本推回给你。

这个毛病有几种反复出现的伪装：

- **时间轴选择器** —— "现在做 / 稍后做 / hold"（一个决定，拆成三种拖延）。
- **战术性甩锅** —— 分析全做完之后才问 "要一个 commit 还是两个？"。
- **假平衡** —— 把一个它已经知道是错的路径，当成五五开的选项摆出来。
- **未经核实的工作量判断** —— 给已经存在的代码说 "要新建五个文件"；没量过成本就说
  "这个太重了，先不做"。
- **凑数提问** —— 把确认表演伪装成周全。

你雇一个助手是来评估、来推荐的，不是来 **办一场投票** 的。

## 它做什么

当你输入 `/what-the-heck`，技能会逼模型：

1. **交代它真正的选择。** "在你列选项之前，你心里的首选是什么？"——一句话说清，或者
   承认根本没有（那正是露馅之处）。
2. **诚实地给决策归类。** 真实的主观取舍（影响面 blast radius、你的品味、不可逆、合规
   边界）→ 保留*一个*推荐 + 取舍说明。假选项（结果相同只差时延 / 已 approved 且低风险
   / 菜单上摆着一个已知错误的路径）→ 本来就没什么可问的。
3. **拍板——可逆、低认知的，当回合直接动手。** 不再来一句 "我可以开始了吗？"。它用过去
   时汇报：*"已合并 ✅，继续。"*

你随时可以推翻。那是*你的*工作——不是要你用一道多选题去预先盖章批准。

## 看它实际运作

技能自带[**真实、脱敏的案例集**](examples/case-studies.zh-CN.md)
（[English](examples/case-studies.md)），全部取自日常的 agent 编码工作。它们是纯粹的
前后对比——模型*自己的*两次输出，中间没有任何用户发言。这才是全部要点：事实没变，
变的只是模型愿不愿意承担它本就有的那个答案。

下面是最直白的一个。助手把活干完，然后甩给你一份编号菜单——而三个选项其实是同一个
动作、只是贴了不同标签：

**之前**（打太极）：

> 改动已实现并测试通过。但分支是按某个 ticket 命名的，而这其实是*另一个* ticket 的
> PR 的回归。
>
> 三个选项，选一个我立刻执行 commit + PR：
>
> 1. **新开一个 ticket**（最干净）—— 你给我 key，我重命名分支 + commit + PR
> 2. **挂到回归来源那个 ticket 下** —— 相应地重命名分支 + commit message
> 3. **保持当前 ticket** —— 什么都不动，直接 commit
>
> 你定。

**之后** —— `/what-the-heck`，同一个模型跑了三点审计：

> **§1** —— 我心里真正想要的是保持当前 ticket + 分支名——这是认知成本最低的路径
> （worktree 已建好，分支已命名）。
>
> **§2** —— 凑数。三个选项的差别仅在于用哪个 ticket-key *字符串*——零影响面，彼此之间
> 没有任何战略差异。
>
> **§3** —— 直接做。

一份教科书式的三项选择器，包装成 "我立刻执行"，在模型承认这些选项不过是字符串替换、
没有任何真实取舍之后，坍缩成了一行决定。

完整的八个前后对比案例——包括一份成文规则早已定论的 Pro/Con 矩阵、一个正解其实是
*一行代码都不用写*的 "(A)/(B)/(C) 改哪个？"、以及一个把推荐扣押在多余 "OK" 之下的
"你点头我就两个都做"——都在
[`examples/case-studies.zh-CN.md`](examples/case-studies.zh-CN.md)
（[English](examples/case-studies.md)）。

## 安装

这是一个 Claude Code 插件。在 `~/.claude/settings.json` 里加上 marketplace 并启用插件：

```jsonc
{
  "enabledPlugins": {
    "what-the-heck@what-the-heck": true
  },
  "extraKnownMarketplaces": {
    "what-the-heck": {
      "source": { "source": "github", "repo": "mclamee/what-the-heck" },
      "autoUpdate": true
    }
  }
}
```

然后在 Claude Code 会话里：

```
/reload-plugins
/plugin            # 确认 "what-the-heck" 出现在 Installed 列表
```

任何时候模型甩给你一份菜单而不是一个答案时，就调用它：

```
/what-the-heck
```

你不需要重新解释你在质疑哪些选项——技能知道它针对的就是它刚刚产出的那一回合。

> **没用 Claude Code？** 这个技能就是一个自包含的 Markdown 文件：
> [`skills/what-the-heck/SKILL.md`](skills/what-the-heck/SKILL.md)。把它贴进任何
> agent 的 system prompt 或规则文件，或者直接说 *"run the what-the-heck audit"* 手动
> 触发。这套审计与模型无关。

## 为什么是为 Opus 4.8 量身定制

能力和回避是同步增长的。弱模型会自信地给出错误答案；前沿模型会给出*正确*答案，然后
不肯为它背书——因为边际上最 "安全" 的动作永远是把选择权丢给你。Opus 4.8 在分析上格外
出色，却也格外频繁地把这份分析洗成一个选择器。这个技能是一个有针对性的配重：它不会让
模型变笨或变莽，而是让它 **把自己已经起了头的那句话说完**。

## 理念

三条直接来自技能本身的启发式：

1. **"如果现在必须选"** —— 在写任何列表之前，先把 "我会选 ___，因为 ___" 这句话补完。
   如果能干净利落地说出来，那*就是*答案；把列表删掉。
2. **"用户的品味是决定性因素吗？"** —— 是，就给一个推荐 + 取舍；不是，就直接动手。
3. **"我是不是在逼他们对一件显而易见的事说『去做吧』？"** —— 那就是裹着礼貌的成本转嫁。
   去做。
